## 📚 Key Concepts Summary

| Concept | What it is | Icon |
|---------|------------|------|
| Docker networks | Private network for containers to talk by name | 🌉 |
| Volumes | Persistent storage that survives container restarts | 💾 |
| Health checks | Verify a service is truly ready before depending on it | ❤️ |
| depends_on | Control startup order based on health | 🔗 |
| Ports | Expose only what needs to be public | 🔌 |
| API Gateway pattern | Single entry point for all client requests | 🚪 |
| Database per service | Each service owns its data, no sharing | 🗄️ |
| RabbitMQ | Async job processing via message queues | 📬 |
| Redis | Cache + rate limiting + service registry | 🔄 |
| Environment variables | Inject config into containers at runtime | ⚙️ |
